# CC 任务监控通知模板

> 用于监控 Claude Code 任务执行状态并自动通知下一个任务的完整流程

---

## 1. cli-agent上下文查找规则

```
上下文文件: {主工作空间}/sessions/cliagent/{sessionKey对应的文件名}.jsonl
```

**查找步骤**:
1. 获取当前 channel 的 sessionKey
2. 根据 sessionKey 查找对应的 .jsonl 文件名
3. 读取该 jsonl 文件的最后部分（20-30行），分析 CC 状态

---

## 2. 状态判断逻辑

```
判断完成标志:
- 最后一条 assistant 消息包含 "✅ 完成" 或 "完成报告"
- 或包含任务汇总表格和验证报告

判断进行中:
- 最后一条消息是工具调用（Read/Grep/Bash等）
- 或包含 "正在执行"、"开始执行" 等关键词
```

---

## 3. 任务通知模板

```markdown
执行以下复刻任务:
{任务ID} 任务,如果存在已有java复刻，请直接认为他复刻不完整

 **执行要求**:

 1. 每个任务必须先动态分解
 2. 每个任务完成后必须验证
 3. 所有任务完成后输出汇总报告
 **输出格式**:
 | 任务ID | 分解项数 | 完成项数 | 验证状态 | 总体状态 |
 |--------|---------|---------|---------|---------|
 | P1-D1-1 | X | X | ✅ | ✅ 完成 |
 | P1-D1-2 | X | X | ✅ | ✅ 完成 |
 | ... | ... | ... | ... | ... |### Step 2: 逐项翻译
 按照分解表格，逐项翻译，每完成一项更新状态为 ✅

 ### Step 3: 功能点覆盖验证
 完成后，输出验证报告:

 | 验证维度 | 源代码数量 | 目标代码数量 | 覆盖率 | 状态 |
 |---------|----------|------------|-------|------|
 | 方法映射 | X | X | 100% | ✅ |
 | 分支覆盖 | X | X | 100% | ✅ |
 | 异常映射 | X | X | 100% | ✅ |
 | 参数验证 | X | X | 100% | ✅ |
 | 日志映射 | X | X | 100% | ✅ |

 **禁止事项**:
 - ❌ 不得跳过动态分解步骤
 - ❌ 不得简化任何分支逻辑
 - ❌ 不得跳过任何异常处理
 - ❌ 不得省略任何参数验证
 - ❌ 不得合并多个方法
```

---

## 4. 定时任务创建

```bash
# 每5分钟执行一次检查
cron add -e 300 -m "检查 CC 任务状态并通知执行"
```

---

## 5. 完整执行流程

```
┌─────────────────────────────────────────────────────────┐
│  Step 1: 读取 sessions.json 获取 sessionKey            │
│  → cli:direct → faint-coral.jsonl                       │
├─────────────────────────────────────────────────────────┤
│  Step 2: 读取上下文文件最后20行                         │
│  → tail -20 faint-coral.jsonl                          │
├─────────────────────────────────────────────────────────┤
│  Step 3: 判断状态                                       │
│  → 完成? → 通知下一个任务                               │
│  → 进行中? → 静默等待                                   │
├─────────────────────────────────────────────────────────┤
│  Step 4: 通知 CC 执行                                   │
│  → cli_agent run -p p1 -m "<任务模板>"                  │
└─────────────────────────────────────────────────────────┘
```

---

## 6. 快速命令参考

### 查看当前 CC 状态
```bash
# 1. 获取 sessionKey
cat sessions/sessions.json

# 2. 读取最新上下文 (最后20行)
read_file offset=140 limit=20 sessions/faint-coral.jsonl
```

### 启动 CC 执行任务
```bash
cli_agent run -p p1 -m "<任务模板>"
```

### 创建定时监控任务
```bash
cron add -e 300 -m "检查 CC 任务状态"
```

---

## 7. 任务 ID 命名规范

```
P{Phase}-D{Day}-{序号}
例如:
- P7-D1-1 = Phase 7, Day 1, 第1个任务
- P7-D1-2 = Phase 7, Day 1, 第2个任务
- P7-D2-1 = Phase 7, Day 2, 第1个任务
```
