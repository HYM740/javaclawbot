## 关键经验

> 压缩后的关键经验

1. **首次使用**：询问作者名、项目路径、数据库连接
2. **场景识别**：根据需求自动识别 A/B/C/D 四种场景
3. **计划管理**：完整需求必须制定计划，逐步执行
4. **回滚机制**：Git 项目用 Git 回滚，非 Git 项目用文件备份
5. **记忆更新**：每次生成后更新记忆，保留关键信息

### 工具使用注意事项

**mysql-tool.js 参数格式**：
- ✅ 正确：`--password "你的密码"`（空格分隔，值用引号包裹）
- ❌ 错误：`--password=你的密码`（等号连接会导致解析失败）
- 原因：密码中包含特殊字符（如 `@`），等号连接会导致参数解析错误
